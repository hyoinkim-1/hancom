# 바닐라 JS 컴포넌트 구현 연습

HTML / CSS / JS 복습을 위해 프레임워크 없이 컴포넌트를 직접 구현해보는 프로젝트.

## 목적

1. HTML, CSS, JS를 복습하는 용도
2. AI를 최소한으로 사용하고, 컴포넌트들을 **바닐라 JS로 직접 구현**해보는 데에 의의
3. CSS는 레이아웃만 직접 잡고, 나머지(컬러 등)는 AI에게 맡길 것

## 요구사항

1. **시맨틱 태그**를 우선적으로 사용할 것
2. 컬러, 폰트, 사이즈는 **CSS 변수**로 관리할 것
3. `@media`를 활용해 **반응형**으로 구현할 것

## 기능 명세

### 1. Header 영역

| # | 기능 | 구현 방식 |
|---|------|-----------|
| 1-1 | 스크롤(드래그)해도 화면 상부에 고정 | `position: sticky` + `top: 0px` |
| 1-2 | `nav` 태그를 담고 있다 | `ul.nav__item` 하위 요소의 `a` 태그로 각 카드 컴포넌트로 이동 |
| 1-3 | 다크모드 토글 버튼 | CSS의 `:root`와 `:root.dark`로 구현 |

### 2. Main 영역

#### 2-1. 레이아웃

- `section.container`에 grid 혹은 flex(`direction: column`) 배치로 카드 컴포넌트들을 화면에 배치
- 구현한 구조들을 카드 컴포넌트에 담아서 화면에 노출
  - 카드 컴포넌트 테두리에 마우스 호버 시 커서가 `pointer`로 변경
  - (optional) 카드 컴포넌트 테두리에서 drag 이벤트로 카드 위치 변경 가능

#### 2-2. 투두 리스트

1. 상단에 text input 창과 그 우측에 `등록` 버튼
2. 등록 버튼 클릭 시 새로운 todo 저장
   - todo-list를 `localStorage`에 저장하고 파싱해서 구현
3. `li` 태그 내의 `삭제` 버튼 클릭 시 해당 todo 삭제

#### 2-3. 캐러셀

1. 오른쪽으로 이미지들이 자동 이동 (마우스 호버 시 정지)
2. 방향 설정 `<` / `>` 버튼 (default: `>`)
3. pause / play 버튼

#### 2-4. 테이블

1. JS 파일 내의 더미 데이터를 표시
2. 1행(헤더)의 제목들에 마우스 호버 시 툴팁 팝업
   - (optional) 툴팁 내용은 정렬 방식을 설명
   - 제목 클릭 시 정렬 방식에 따라 테이블 내용 정렬
