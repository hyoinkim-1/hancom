}
정답입니다! fr은 gap을 제외한 영역만 분할합니다.
오답입니다. 정답은 ②번입니다.
해설 보기

% 대신 fr을 사용하는 이유는 gap 처리 때문입니다.

  • %: gap을 포함한 전체 너비 기준으로 계산 → 부정확
  • fr: gap을 제외한 실제 콘텐츠 영역 기준 계산 → 정확

5/5
정답입니다! LVHA 순서(코드 A)가 정답입니다.
오답입니다. 정답은 ①번입니다.
해설 보기

CSS는 나중에 작성된 규칙이 먼저 작성된 규칙을 덮어씁니다. 가상클래스는 반드시 LVHA(Link → Visited → Hover → Active) 순서로 작성해야 :hover, :active가 의도대로 동작합니다.

a:link { color: blue; }       /* 1순위 */
a:visited { color: purple; }  /* 2순위 */
a:hover { color: red; }       /* 3순위 */
a:active { color: orange; }   /* 4순위 (가장 최신) */
