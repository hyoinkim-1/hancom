- Day 13 `2026.07.15 수` vercel
    - vercel 전역으로 설치한 후 터미널로 배포
        - `npm  i -g vercel`
        - 버셀 인증 `vercel login`
        - `vercel --prod` 프로덕션환경으로 배포
        - 풀스택으로 배포할시 `vercel.json` 파일 생성 후 services 추가한후 client와 server에 대한 설정을 추가적으로 해줘야함. 기존 경로들을 상대경로로 바꾸면 됨.
            
            ```json
            {
              "services": {
                "client": {
                  "root": "client/"
                },
                "server": {
                  "root": "server/",
                  "entrypoint": "server.js"
                }
              },
              "rewrites": [
                { "source": "/api/(.*)", "destination": { "service": "server" } },
                { "source": "/(.*)", "destination": { "service": "client" } }
              ]
            }
            ```
            
        - 로컬에서 테스트해보기 위해선 client/vite.config.ts 파일에서  server.proxy 설정 추가로  localhost 프록시 추가
            
            ```tsx
            import { defineConfig } from 'vite'
            import react from '@vitejs/plugin-react'
            
            // https://vite.dev/config/
            export default defineConfig({
              plugins: [react()],
              server: {
                proxy: {
                  '/api': 'http://localhost:3000',
                },
              },
            })
            
            ```
